## بنية ملف CSV

الملف يحتوي على صف لكل آية في القرآن الكريم (6236 صف).

| العمود | النوع | الوصف | مثال |
|---|---|---|---|
| `اسم السورة` | نص | الاسم العربي للسورة | `البقرة` |
| `ترتيب السورة` | رقم صحيح (1–114) | رقم السورة في المصحف | `2` |
| `رقم الآية` | رقم صحيح | رقم الآية داخل السورة | `255` |
| `كم تمثل الآية من سطر` | عشري | حجم الآية بالأسطر | `5.0` |

### قواعد قيمة السطر

- **آية قصيرة** → قيمة أقل من 1 (مثال: `0.5`)
- **آية عادية** → من 1 إلى 3 تقريباً
- **آية طويلة** → قد تصل 15 (مثال: آية الدَّين `2:282 = 15.0`)
- **المجموع الكلي** للقرآن ≈ 604 صفحة × 15 سطر = **~9060 سطراً**

### مثال على بيانات صحيحة

```csv
اسم السورة,ترتيب السورة,رقم الآية,كم تمثل الآية من سطر
الفاتحة,1,1,2
الفاتحة,1,2,0.7
الفاتحة,1,3,0.6
...
البقرة,2,1,1.3
البقرة,2,2,1.5
```

## ما يفعله السكربت

### 1. قراءة CSV وترتيبه

```
CSV
 ↓
المسار الأمامي  (الفاتحة → الناس): مرتّب تصاعدياً بـ surah_order ثم ayah_num
المسار العكسي  (الناس → الفاتحة): مرتّب تنازلياً بـ surah_order، تصاعدياً بـ ayah_num
```

### 2. بناء Prefix Sum Array

لكل مسار، يُحسب الـ cumulative sum للأسطر:

```
الآية  | الأسطر | التراكمي
-------+---------+---------
1:1    |  2.0   |  2.0
1:2    |  0.7   |  2.7
1:3    |  0.6   |  3.3
...
```

هذا يُتيح استعلام O(1) لأي نطاق: `lines(from, to) = cum[to] - cum[from-1]`

### 3. بناء Index Maps

```
"1:1" → 0
"1:2" → 1
"1:3" → 2
...
"114:6" → 6235
```

تحويل فوري O(1) من `سورة:آية` إلى موقع في المصفوفة.

### 4. بناء Reverse Index

مصفوفة عكسية للتحويل السريع O(1) من موقع إلى `سورة:آية`:

```
index 0    → { surah: 1,   ayah: 1 }
index 1    → { surah: 1,   ayah: 2 }
...
index 6235 → { surah: 114, ayah: 6 }
```

### الناتج النهائي في TypeScript

```typescript
// للمسار الأمامي (الفاتحة → الناس):
RAW_CUMULATIVE_ARRAY_FORWARD  // Float32Array — الأسطر التراكمية
INDEX_MAP_FORWARD              // Record<string, number> — "سورة:آية" → index
REVERSE_INDEX_FORWARD          // Array — index → { surah, ayah, is_end }

// للمسار العكسي (الناس → الفاتحة):
RAW_CUMULATIVE_ARRAY_REVERSE
INDEX_MAP_REVERSE
REVERSE_INDEX_REVERSE

// بيانات السور:
SURAH_NAMES   // string[] — الأسماء العربية (114 عنصر)
SURAH_INFO    // [string, number][] — [الاسم، عدد الآيات]
```

## تعديل البيانات

### تصحيح قيمة سطر لآية معينة

افتح ملف CSV وعدّل القيمة في العمود الرابع:

```csv
# قبل
البقرة,2,282,14

# بعد
البقرة,2,282,15
```

ثم أعد تشغيل السكربت لتوليد `QuranStaticData.ts` جديد.

### إضافة مصحف مختلف

يكفي إنشاء CSV جديد بنفس البنية (قيم الأسطر تختلف حسب المصحف)، ثم تشغيل السكربت عليه.

---

## لماذا Float32Array وليس Number[]؟

| النوع | الحجم لكل عنصر | إجمالي 6236 عنصر |
|---|---|---|
| `Number[]` | 8 bytes | ~50 KB |
| `Float32Array` | 4 bytes | **~25 KB** |

توفير 50% في الذاكرة مع دقة كافية لقيم الأسطر (أقصى خطأ تقريب: 0.0001 سطر).
